# 執筆設計書

## 執筆アプローチ

ハンズオン形式の実践章。読者が実際に手を動かしてサンプルアプリケーションをデプロイできるように、コード例・コマンド例を豊富に含める。ただし概念理解に必要な最小限のコードに絞り、完全なマニフェストはリポジトリ参照とする。

## 節ごとの設計

### 1.1 サンプルアプリケーションの全体像

**導入**: どのようなアプリケーションを構築するかの全体像を示す

**図表設計**:
- 図1.1: Mermaidで4サービス+DBのアーキテクチャ図。リクエストフローを矢印で表現
- 表1.1: サービス名、役割、言語、ポート番号を表形式で整理

**本文の構成**:
1. アプリケーション概要（ECサイトの簡易版）
2. 各サービスの役割説明
3. サービス間通信フロー
4. Goコード抜粋（APIゲートウェイ）

### 1.2 Kubernetesマニフェストの作成

**導入**: 各サービスをK8sリソースとしてどう定義するか

**本文の構成**:
1. Namespaceの作成
2. Deploymentの定義（代表例として商品サービス）
3. Serviceの定義
4. ConfigMapの定義
5. Ingressの定義

### 1.3 Kustomizeの基本 ― base + overlays

**導入**: なぜKustomizeが必要かを環境差分の問題から導入

**図表設計**:

4. kustomize buildの実行と結果確認

### 1.4 サンプルアプリケーションのデプロイと動作確認

**導入**: 実際にデプロイして動かす
